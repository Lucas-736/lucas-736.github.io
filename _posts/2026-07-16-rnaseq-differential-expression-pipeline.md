差异表达（Differential Expression, DE）分析是转录组里最常跑的流程。第一次接触时，最容易被淹没在一堆工具名里（FastQC、STAR、salmon、DESeq2……），却说不清它们各自解决什么问题。这篇把主干拆开，讲清楚**每一步在做什么、为什么需要它**。

## 主干一览

```
fastq → 质控 → 比对/定量 → 表达矩阵 → 差异分析 → 功能富集
```

一条 RNA-seq 数据从原始测序到「哪些基因显著变化、变化意味着什么」，中间就是这几步。下面逐段说。

## 1. 质控（QC）

拿到 `fastq` 先别急着比对。用 **FastQC** 看每个样本的碱基质量、接头污染、GC 分布、重复率；多个样本用 **MultiQC** 汇总成一张报告，一眼看出哪个样本异常。

这一步的意义是：**垃圾进，垃圾出**。测序质量差、接头没切干净，后面所有结论都不可靠。

## 2. 去接头 / 质量修剪

根据 QC 结果，用 **fastp**（快、一步到位）或 Trimmomatic 切掉接头和低质量碱基。数据干净的话这步影响不大，但一旦有明显接头污染，就必须做。

## 3. 比对 or 定量：两条路线

这里有个关键分叉：

- **传统比对路线**：用 **HISAT2 / STAR** 把 reads 比对到参考基因组，再用 **featureCounts** 数每个基因上落了多少 reads。优点是能拿到 BAM、可视化、做可变剪接等下游分析。
- **伪比对（pseudo-alignment）路线**：用 **salmon / kallisto** 直接在转录本层面定量，跳过生成 BAM，**又快又省内存**，还能校正 GC/序列偏差。配合 `tximport` 汇总到基因层面。

如果你只关心基因层面的差异表达，salmon 路线通常是性价比更高的选择。

## 4. 得到表达矩阵

无论哪条路线，最终都得到一张 **基因 × 样本** 的计数矩阵（counts）。**注意：差异分析要用原始 counts，不要自己先转成 TPM/FPKM 再丢进 DESeq2**——DESeq2 需要原始计数来建模离散度，喂归一化后的值会破坏它的统计假设。

## 5. 差异表达分析

主流工具是 **DESeq2** 和 **edgeR**，都基于负二项分布建模计数数据。核心产出是每个基因的：

- **log2FoldChange**：变化幅度与方向
- **p 值 / padj**：显著性（padj 是多重检验校正后的）

筛差异基因时，别只卡 p 值，要用 **padj（FDR）** + 一个 fold change 阈值。至于为什么是负二项分布、DESeq2 又做了哪些「小样本救命」的处理，我单独写了一篇：[读懂 DESeq2](/posts/why-deseq2-negative-binomial/)。

## 6. 功能富集

拿到一串差异基因只是中间产物，真正要回答的是「**它们在生物学上意味着什么**」。这一步用 **clusterProfiler** 做 GO / KEGG 富集，把基因列表翻译成通路和功能。

富集又分 ORA 和 GSEA 两种思路，选错了结论会有偏差——这个话题我也单独拆了一篇：[ORA 还是 GSEA](/posts/ora-vs-gsea-enrichment/)。

## 小结

整条流程的逻辑其实很清晰：**保证数据干净 → 数出每个基因的表达量 → 统计出谁在显著变化 → 把基因翻译成生物学意义**。工具会换、会更新，但这条主干很稳定。把每一步「为什么存在」想明白，比记住某个命令行参数重要得多。
