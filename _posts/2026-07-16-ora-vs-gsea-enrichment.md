差异分析拿到基因列表后，下一步是富集分析，把基因翻译成通路和功能。但「富集」其实有两种完全不同的做法——**ORA** 和 **GSEA**。它们的假设不一样，用错了会得出有偏差的结论。这篇把区别讲透。

## ORA：过表征分析

**ORA（Over-Representation Analysis）** 是最直觉的做法：

1. 先卡个阈值（比如 `padj < 0.05 & |log2FC| > 1`），拿到一份**显著差异基因名单**；
2. 对某个基因集（如某条 GO term），问：**「这份名单里落在该基因集的基因，是不是比随机预期得多？」**
3. 用**超几何分布 / Fisher 精确检验**算 p 值。

`clusterProfiler` 里对应 `enrichGO()` / `enrichKEGG()`。

**它的局限：**

- **强依赖那个人为阈值**。padj 卡 0.05 还是 0.01，名单不同，结论就可能不同。
- **只看「在不在名单里」，丢掉了排序信息**。一个刚好没过线的基因（padj = 0.051）被完全忽略。
- **抓不住「协同的微小变化」**。如果一条通路上几十个基因每个都只轻微上调、单独都不显著，ORA 会全部漏掉。

## GSEA：基因集富集分析

**GSEA（Gene Set Enrichment Analysis）** 换了个思路：**不卡阈值，用全部基因**。

1. 把**所有**基因按某个统计量（如 log2FC 或检验统计量）**从高到低排序**；
2. 沿着这个排序列表走，遇到目标基因集里的基因就加分、不在的就扣分，得到一条**running sum**，峰值即**富集得分 ES**；
3. 通过置换检验判断这个 ES 是否显著。

`clusterProfiler` 里对应 `gseGO()` / `gseKEGG()`。

**它的优势：**

- **不需要人为阈值**，避免了「卡在哪」的主观性；
- **保留了完整排序信息**；
- **能检测协同的微小变化**——一整条通路都轻微上调时，这些基因会集中在排序列表顶端，ES 就会显著。这正是 ORA 的盲区。

## 怎么选

| | ORA | GSEA |
|---|---|---|
| 输入 | 显著基因名单 | 全部基因 + 排序值 |
| 依赖阈值 | 是 | 否 |
| 利用排序 | 否 | 是 |
| 擅长 | 强信号、名单明确 | 协同的微小变化 |

实用建议：

- 已经有一份**明确、可信的差异基因名单**，想快速看它们富集在哪 → **ORA**。
- 想要**更全面、少受阈值主观影响**，尤其怀疑存在「整条通路轻微变化」 → **GSEA**。
- 两个都跑、互相印证，往往是最稳的做法。

## 一个常被忽略的细节

GSEA 的排序统计量选什么很关键。只用 log2FC 会让**低表达但倍数虚高**的基因排到前面；结合显著性的统计量（如 `stat` 或 `-log10(p)·sign(log2FC)`）通常更稳。**排序的质量，直接决定 GSEA 结果的质量。**

## 小结

ORA 问的是「**我这份名单里，哪条通路超标了**」；GSEA 问的是「**在整个排序里，哪条通路整体偏向一端**」。前者简单直接但受阈值摆布、会漏掉协同弱信号；后者更全面但依赖好的排序。搞清楚这层区别，富集结果才不会误导你。
